# 玩家指南

本文说明 HorrorCooked9 中与烹饪、加工相关的主要玩法与操作习惯；具体数值以游戏内配方与配置为准。

## 沙拉碗（搅拌与取餐）

- 将沙拉碗放在合适位置后，按配方投入食材与盛装容器，使用**搅拌工具**完成搅拌小游戏（节奏与平衡受数据配置影响）。
- 从碗中**取走成品**会消耗配方中的盛装容器（多为碗）。
- **吃完**对应沙拉成品后，通常会**返还盛装容器**（与配方一致）；搅拌工具**不会**因此返还，详见 [salad-eat-return.md](salad-eat-return.md)。
- 沙拉碗在成品状态下会有相应展示（模型与摆放表现随版本迭代可能调整）。

## 砧板与切割小游戏

- **砧板**必须放置在 **Foodworks 桌**正上方才会启用完整交互。
- **放置食材**：手持可放置物品（见数据标签 `chopper_placeable`），**按住潜行（Shift）** 右键砧板放置一格物品；主手持**菜刀**时，会优先把**副手**中可放置物品放到砧板上（便于双手配合）。
- **取下食材**：砧板上有物品时，**潜行**右键可取下（无物品时潜行不会误触）。
- **开始切割**：砧板上有物品且存在对应 `chopper_minigame` 配方时，主手**菜刀**右键可打开**切割小游戏**；完成小游戏后按配方产出。

## 手持砧板：格挡与冲刺

- 将**砧板**拿在主手或副手，**不按潜行**时使用物品，会进入类似举盾的长按姿态（可用于格挡）。
- 在该姿态下按下**攻击键**（默认左键），会向面朝方向**短距离冲刺**，路径上的敌对生物会受到少量伤害；冲刺有**冷却时间**，避免连续滥用。
- **潜行**时使用物品则按普通**方块放置**逻辑，避免与冲刺混淆。

## 磨刀石与菜刀

- **磨刀石**用于修理或维护**菜刀**（具体交互与消耗见游戏内说明与 `sharpening_stones` 数据配置）。
- 菜刀是砧板玩法与部分交互的核心工具，耐久与修理方式以实际版本为准。

## 指南书（Patchouli）

- 模组提供基于 **Patchouli** 的**指南书**，内含入门分类与条目（中英资源若均已提供，可在游戏内切换语言查看）。
- 若未安装 Patchouli，相关物品或书页可能无法使用，请确认整合包或服务器已包含该依赖。

## 榨汁机

- **榨汁机**用于将指定水果等原料加工为流体（如菠萝汁），并可装瓶或与其他系统联动。
- 机器有容量、手动/自动步骤等设定，详见默认数据 `gameplay/juicing.json`；副产物（如糖、骨粉）由配置条目决定。

## 简易过滤器与清水

- **简易过滤器**用于将「清水」类流体处理为「过滤水」等（具体以标签与方块说明为准）。
- **过滤水**可在熔炉、高炉、烟熏炉、篝火等中**煮沸**为可装桶的清水类产物（配方与容器标签见数据包）；部分场景下也可与**炼药锅**加热规则联动，见 `clear_water_boiling.json`。

## 屠宰与尸体

- 使用配置中的**屠宰武器**击杀指定牲畜后，可能生成**尸体物品**并替换或抑制部分原版生肉掉落（以 `butchery.json` 为准）。
- 手持尸体时，副手需按配置持有**收割工具**，对尸体进行收割可获得内脏等掉落；收割有冷却并对工具有损耗。
- 许多内脏与食材的**饱食度、效果**由 `item_foods.json` 单独配置，可在数据包中覆盖。

## 数据包与服务器（服主与玩家须知）

以下命令仅在**专用服务器**上生效，且受服务端配置开关控制；单人本地一般不可用。

| 命令 | 权限 | 作用 |
| --- | --- | --- |
| `/datapack upload` | 2 级及以上 | 向服务器世界数据包目录上传 zip 或文件夹（客户端会弹出选择器） |
| `/datapack delete <名称>` | 2 级及以上 | 删除已上传的数据包目录；删除后通常需要 `/reload` 等操作使游戏重新加载数据 |

若服主在配置中关闭了数据包上传相关功能，上述命令会提示已禁用。

## 其他常用命令（管理向）

| 命令 | 说明 |
| --- | --- |
| `/identity give ...` | 按 JSON 规则给带指定 tag 的在线玩家分配身份，见 [identity_usage_manual.md](identity_usage_manual.md) |
| `/random value <范围>` | 在指定整数范围内输出随机数，范围写法支持 `5..10`、`..10`、`5..`、单个数或两个空格分隔的数 |

若你与数据包作者协作，修改世界内数据后务必确认服务器与客户端资源一致，并在允许时执行重载或重启。
